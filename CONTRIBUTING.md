# 貢献ガイド

このプロジェクトへの貢献ありがとうございます！

## 始める前に

1. **リポジトリをフォーク**（または直接クローン）
2. **最新版をプル**
   ```bash
   git pull origin main
   ```
3. **自分のブランチを作成**
   ```bash
   git checkout -b feature/あなたの機能
   ```

## 自動コミットについて

⚠️ **このリポジトリは GitHub Actions で 10分ごとに自動コミットされます**

### 何が起きているか
- GitHub がリポジトリ内の変更を監視
- 10分ごとに変動があれば `Auto-commit: ...` というメッセージでプッシュ
- 全チームメンバーの変更が対象

### 注意点
- 自動コミットはすべての編集者に影響
- コミット履歴が多くなる可能性がある
- 手動で `git push` する前に `git pull` して最新化を推奨

## コミットメッセージのルール

わかりやすいメッセージを付けてください：


### 良い例
- `Add: 問題1-5を追加`
- `Fix: 解答の誤字を修正`
- `Update: 問題の難易度を調整`
- `Refactor: 構成を整理`

### 避けるべき例
- ❌ `update`
- ❌ `bug fix`
- ❌ `修正`

## プッシュ前にチェック

- [ ] ファイルは保存した？
- [ ] ビルドエラーがない？
- [ ] 他の人の変更と重複していない？（`git pull` で確認）

## 大きな変更の場合

1. 先に `memo` に進捗を記録
2. Discord/Slack で相談
3. 完成後にプッシュ

## 問題が発生した場合

- GitHub Issues でバグを報告
- 詳しく「何をしたか」「何が起きたか」を書く
